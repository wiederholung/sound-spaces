# Standalone Audio Processor for Sound Spaces

This repository contains a standalone Python implementation of the sound processing pipeline from the `sound-spaces` repository. The `StandaloneAudioProcessor` allows you to process impulse responses with source audio to generate spatialized audio observations independently of the full sound-spaces framework.

## Overview

The standalone processor replicates the exact audio processing workflow from the original sound-spaces implementation:

1. **Load source audio** - Load any audio file as the source sound
2. **Process impulse responses** - Convolve source audio with pre-computed impulse responses 
3. **Apply crossfading** - Smooth transitions between consecutive audio observations
4. **Generate complete audio** - Concatenate all steps into a complete spatialized audio track

This is useful when you have:
- Pre-computed impulse responses from acoustic simulations
- Source audio that was playing during an episode  
- Need to generate what an agent would have heard during navigation

## Quick Start

### Basic Usage

```python
from standalone_audio_processor import StandaloneAudioProcessor

# Initialize processor
processor = StandaloneAudioProcessor(
    sampling_rate=16000,
    step_time=0.1,
    crossfade_enabled=True,
    crossfade_duration=0.05
)

# Load your source audio
source_audio = processor.load_source_audio("path/to/source.wav")

# Load your impulse responses (list of numpy arrays)
# Each IR should have shape (ir_length, num_channels)
impulse_responses = [
    # Your IRs here - one per navigation step
    # np.load("step_0_ir.npy"),
    # np.load("step_1_ir.npy"), 
    # ...
]

# Process audio
step_observations, complete_audio = processor.process_impulse_responses(
    source_audio, impulse_responses
)

# Save results
processor.save_audio(complete_audio, "output_spatialized.wav")
```

### What You Get

1. **Step-by-step observations** - List of convolved audio for each navigation step
   - Each observation has shape `(num_channels, samples_per_step)`
   - Represents what the agent heard during that specific step

2. **Complete concatenated audio** - Full spatialized audio for entire episode
   - Shape: `(num_channels, total_samples)`
   - Represents the complete audio experience

## Parameters

### StandaloneAudioProcessor

- `sampling_rate` (int): Audio sampling rate in Hz (default: 16000)
- `step_time` (float): Duration of each step in seconds (default: 0.1)  
- `crossfade_enabled` (bool): Enable crossfading between steps (default: True)
- `crossfade_duration` (float): Crossfade duration in seconds (default: 0.05)

### Input Requirements

**Source Audio:**
- Any audio format supported by librosa
- Will be resampled to the specified `sampling_rate`
- Should be long enough to cover all navigation steps

**Impulse Responses:**
- List of numpy arrays, one per navigation step
- Each IR should have shape `(ir_length, num_channels)`
- Typically stereo (2 channels) for binaural audio
- Can be mono (1 channel) - will be handled appropriately

## Integration with Sound Spaces

This standalone processor is designed to work with impulse responses generated by the sound-spaces framework:

1. **Generate IRs** using sound-spaces acoustic simulation
2. **Save IRs** for each navigation step 
3. **Use this processor** to generate spatialized audio offline

### Example Integration

```python
# After running sound-spaces simulation and collecting IRs
impulse_responses = []
for step in range(num_steps):
    # Load IR from your sound-spaces simulation
    ir = np.load(f"sound_spaces_output/step_{step}_ir.npy")
    impulse_responses.append(ir)

# Process with source audio
processor = StandaloneAudioProcessor()
source_audio = processor.load_source_audio("source_sound.wav")
step_obs, complete = processor.process_impulse_responses(source_audio, impulse_responses)
```

## Technical Details

### Convolution Algorithm

The processor uses the same convolution algorithm as the original implementation:

1. **Temporal windowing** - Extract appropriate audio segment for current step
2. **FFT convolution** - Efficient convolution using `scipy.signal.fftconvolve`
3. **Reverb handling** - Include reverb tail from previous time steps
4. **Wraparound support** - Handle audio looping for long episodes

### Crossfading

Optional crossfading between consecutive steps:

1. **Overlap region** - Short overlap between consecutive audio segments
2. **Linear interpolation** - Smooth transition using weighted average
3. **Configurable duration** - Adjust crossfade length as needed

### Output Format

- **Stereo audio** - 2-channel output for binaural spatialization
- **Consistent timing** - Each step has exactly `step_time` duration
- **High quality** - Preserves full audio fidelity during processing
